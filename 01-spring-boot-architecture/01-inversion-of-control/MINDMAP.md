# Inversion of Control and Dependency Injection Mindmap

- Inversion of Control (IoC)
  - Problem
    - Tight coupling from `new`
    - Hard to unit test
    - Hard to swap implementations
  - Solution
    - Move object creation into Spring
    - Depend on interfaces and abstractions
  - Application Context
    - Bean registry
    - Lifecycle manager
    - Singleton container by default
- Dependency Injection
  - Constructor injection
    - Preferred enterprise style
    - Immutability with `final`
    - Easy to test
  - Setter injection
    - Optional dependencies
    - Mutable and easier to misuse
  - Field injection
    - Anti-pattern
    - Hidden dependencies
- Bean Lifecycle and Scopes
  - Instantiation
  - Dependency wiring
  - `@PostConstruct`
  - Singleton scope
  - Prototype scope
  - Request and session scope
- Component Scanning
  - `@SpringBootApplication`
  - `@Component`
  - `@Service`
  - `@Repository`
  - `@Controller`
  - `@Configuration` and `@Bean`
