# Module 4: Memory Management & Exceptions

## Overview
This module transitions from basic syntax to JVM architecture and system stability. You will understand where your data physically lives (Stack vs. Heap), how it dies (Garbage Collection), and how to elegantly handle system failures via Exceptions.

## Core Concepts
- **Memory Architecture:** Stack (Thread-Local) vs Heap (Shared).
- **Garbage Collection:** GC Roots, Reachability, and the Minor/Major GC phases.
- **Exceptions:** Error vs Exception, Checked vs Unchecked mapping.
- **Resource Management:** Utilizing `try-with-resources` to prevent catastrophic IO memory leaks. 

## Structure
- `explanation/`: Markdown files and Java Demos explaining memory and exception concepts.
- `exercises/`: Hands-on challenges covering parsing, `try-catch-finally` flows, and defining robust Custom Exceptions.
