# Module 5: Collections Framework & Data Structures

## Overview
This module explores the Java Collections Framework natively, focusing on how architects choose Data Structures based precisely on Big O complexity, Thread-Safety constraints, and Memory locality. Understanding this hierarchy fundamentally shifts a generic Java developer into an engineering architect.

## Core Concepts
- **Collection vs Map:** The fundamental mathematical divide in the interface trees.
- **Lists (`ArrayList`, `LinkedList`):** Index-based sequential data access.
- **Sets (`HashSet`, `TreeSet`):** Uniqueness constraints via Hashing and Tree structures.
- **Maps (`HashMap`, `ConcurrentHashMap`):** Dictionary key-value pairings with massive performance capabilities (and dead-locking dangers).
- **Queues/Deques (`PriorityQueue`, `ArrayDeque`):** FIFO, LIFO, and priority-driven structurally pipelines.
- **Sorting:** Distinguishing intrinsic sorting (`Comparable`) vs extrinsic sorting (`Comparator`).

## Structure
- `explanation/`: Deep dives into the structural interfaces natively.
- `exercises/`: Challenges utilizing sets for uniqueness and priority queues for sorting algorithms.
